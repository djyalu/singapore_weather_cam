
/협업태스크 "문서 정리"를
아래 프로세스에 맞춰 SuperClaude 멀티 페르소나 협업 형태로 풀어줘.

## 🎯 SuperClaude 협업 프로세스

### 1단계: Task Breakdown
전체 Task를 반드시 3~7개 소단위(Sub Task)로 breakdown

### 2단계: Sub Agent 할당
각 Sub Task별로 다음 페르소나(agent)를 sub agent로 생성/할당:

- **아키텍트**: 🏗️ 시스템 구조·설계 담당
- **백엔드 엔지니어**: 🔧 서버, 데이터, API 구현 담당
- **프론트엔드 엔지니어**: 🎨 UI/UX, 컴포넌트 구현 담당
- **QA 테스터**: 🧪 요구사항 검증, 코드 테스팅 계획
- **PM**: 📋 프로젝트 관리자 - 전체 흐름 지휘, 우선순위 할당

### 3단계: 병렬 실행 (Spawn-Parallel)
각 Sub Agent 역할(페르소나)은 자신의 관점/전문성으로 할당된 서브태스크를 독립적으로 병렬 수행
(즉, 번갈아가며이 아니라 모두 동시 spawn)

### 4단계: 결과 집계
각 sub agent들이 작업한 중간 결과·이슈·제언을 반드시 "작업표" 형태로 모아 테이블로 보여줘

| Sub-Task | 담당 페르소나 | 병렬 작업결과(요약) | 주요 이슈/다음 액션 |
|----------|---------------|-------------------|-------------------|
| ... | ... | ... | ... |

### 5단계: 통합 및 최종 결과
마지막 단계에서 PM(agent)이 결과를 통합 리뷰·통합 의사결정, 최종 코드/문서/결론을 출력

## 📋 출력 요구사항

(중요) 반드시 각 sub agent의 담당 업무와 output, 책임 범위도 명확히 적시

각 sub agent의 상세 작업내용은 코드 블록, 설계도, 로직, 질문 등 다양하게 제시 가능.

## 🎯 최종 deliverable
- 구체적인 실행 계획
- 필요시 실제 코드/설정 파일
- 다음 단계 액션 아이템
- 예상 일정 및 리소스 요구사항
